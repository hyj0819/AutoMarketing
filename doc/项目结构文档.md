# AutoMarketing 项目结构说明文档

## 一、项目概述

### 1.1 项目简介

AutoMarketing 是一个基于 Python + Playwright 的社交媒体自动化营销系统，包含**自动化爬虫模块**和**后台管理系统**两部分：

| 模块 | 功能 | 技术栈 |
|------|------|--------|
| **自动化爬虫** | 多平台内容爬取、AI 意图识别、自动化私信触达 | Python + Playwright + DeepSeek |
| **后台管理系统** | 配置管理、数据管理、统计分析、任务监控 | FastAPI + SQLAlchemy + SQLite |

### 1.2 核心特性

| 特性 | 说明 |
|------|------|
| **多平台支持** | 支持 9 个主流社交媒体平台 |
| **AI 意图识别** | 基于 DeepSeek 大模型自动判断潜在客户 |
| **拟人化行为** | 模拟真人滚动、点击、停顿，降低封号风险 |
| **多业务线** | 支持高尔夫模拟器、股票等多条业务线独立运行 |
| **去重机制** | 自动记录已触达用户，避免重复联系 |
| **反爬虫对抗** | 使用指纹浏览器、禁用自动化特征等策略 |
| **后台管理** | 完整的 RESTful API，支持配置管理、数据查询、统计分析 |
| **AI 模型配置** | 可插拔的 AI 模型配置，支持 DeepSeek、OpenAI、Anthropic 等 |

### 1.3 业务场景

| 业务线 | 目标客户 | 主要平台 |
|--------|----------|----------|
| **高尔夫模拟器** | 对室内高尔夫、launch monitor 感兴趣的用户 | Reddit, TikTok, Twitter |
| **股票交易** | 对股票投资、交易策略感兴趣的用户 | Reddit, TikTok, Twitter, TradingView |

### 3.2 src/core/ — 核心配置

#### 3.2.1 database.py — 数据库连接

```python
"""
SQLite 数据库连接配置
- WAL 模式优化写入性能
- 内存映射 1GB
- 缓存大小 64MB
"""

def configure_sqlite_connection(dbapi_connection, connection_record):
    """配置 SQLite 连接优化"""
    cursor = dbapi_connection.cursor()
    cursor.execute('PRAGMA journal_mode=WAL')
    cursor.execute('PRAGMA synchronous=NORMAL')
    cursor.execute('PRAGMA cache_size=-16384')
    cursor.execute('PRAGMA mmap_size=1073741824')
    cursor.execute('PRAGMA temp_store=MEMORY')
    cursor.execute('PRAGMA foreign_keys=ON')
    cursor.close()
```

#### 3.2.2 security.py — 安全模块

```python
"""
API Key 加密模块
- 使用 Fernet 对称加密
- 加密密钥来自环境变量或自动生成
- 支持 API Key 脱敏显示
"""

def encrypt_api_key(api_key: str) -> str:
    """加密 API Key"""
    if not api_key:
        return ''
    return fernet.encrypt(api_key.encode()).decode()

def decrypt_api_key(encrypted_key: str) -> str:
    """解密 API Key"""
    if not encrypted_key:
        return ''
    return fernet.decrypt(encrypted_key.encode()).decode()

def mask_api_key(api_key: str) -> str:
    """脱敏显示 API Key"""
    if not api_key or len(api_key) <= 8:
        return '******'
    prefix = api_key[:4]
    suffix = api_key[-4:]
    return f"{prefix}******{suffix}"
```

**业务流程：**
```
scrape_articles_v0 → 按关键词搜索 Subreddit，滚动加载帖子列表
      ↓
scrape_reviews_v0 → 进入帖子详情页，递归加载并解析所有评论
      ↓
is_target_user() → 调用 DeepSeek API 判断评论者是否为潜在客户
      ↓
chat_v0 → 向目标用户发送私信，记录已触达用户
```

## 九、配置文件说明

### 9.1 环境变量配置（.env）

> 注意：`.env` 文件未提交到版本控制，需在项目根目录手动创建。

```env
# 数据库
DATABASE_URL=sqlite:///./automarketing.db

# API 配置
API_HOST=0.0.0.0
API_PORT=8000
API_DEBUG=true

# CORS 配置
CORS_ORIGINS=http://localhost:8001,http://localhost:3000

# 安全配置 - API Key 加密密钥
# 生产环境请使用 openssl rand -hex 32 生成随机密钥
AES_ENCRYPTION_KEY=auto_marketing_dev_key_change_in_production

# 爬虫配置
CRAWLER_DELAY_MIN=2
CRAWLER_DELAY_MAX=5
CRAWLER_MAX_RETRY=3
```

### 9.2 配置项说明

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `DATABASE_URL` | SQLite 数据库连接路径 | `sqlite:///./automarketing.db` |
| `API_HOST` | API 服务监听地址 | `0.0.0.0` |
| `API_PORT` | API 服务监听端口 | `8000` |
| `API_DEBUG` | 调试模式开关 | `true` |
| `CORS_ORIGINS` | 允许的跨域源（逗号分隔） | `http://localhost:8001,http://localhost:3000` |
| `AES_ENCRYPTION_KEY` | API Key 加密密钥（32字节） | `auto_marketing_dev_key_change_in_production` |
| `CRAWLER_DELAY_MIN` | 爬虫最小延迟（秒） | `2` |
| `CRAWLER_DELAY_MAX` | 爬虫最大延迟（秒） | `5` |
| `CRAWLER_MAX_RETRY` | 爬虫最大重试次数 | `3` |

## 十一、注意事项

### 11.1 登录态维护

- 首次运行需手动登录，后续通过 `launch_persistent_context` 自动保持
- 不同业务线使用不同的浏览器数据目录
- TikTok 需添加 `--disable-blink-features=AutomationControlled` 参数

### 11.2 反爬虫策略

- **禁用自动化特征**：`--disable-blink-features=AutomationControlled`
- **拟人化行为**：随机滚动、随机停顿、模拟点击
- **指纹浏览器**：使用 AdsPower 多账号管理
- **会话刷新**：TikTok 定期回首页刷会话状态

### 11.3 私信限制

| 平台 | 限制 | 应对策略 |
|------|------|----------|
| **TikTok** | 字符长度限制 350 字符 | 控制私信长度 |
| **TikTok** | 发送过快触发限制 | 随机停顿、限流检测 |
| **TikTok** | 部分用户关闭私信 | 回复评论作为备选方案 |
| **Reddit** | 容易被封号 | 控制频率、使用多账号 |
| **TradingView** | 私信数量限制 | 多账号、多脚本并发 |

### 11.4 数据库优化

- 使用 WAL 模式提升写入性能
- 内存映射 1GB 加速读取
- 64MB 缓存减少磁盘 IO
- 外键约束保证数据完整性

### 11.5 Python 版本兼容性

- 项目使用 `.venv` 虚拟环境
- 推荐使用 Python 3.9+ 版本运行
