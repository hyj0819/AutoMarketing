### 4.2 数据库设计

#### 4.2.1 数据库选型说明

**SQLite vs MySQL 对比分析：**

| 对比维度 | SQLite | MySQL | 本项目选择 |
|----------|--------|-------|------------|
| **类型** | 嵌入式文件型数据库 | 客户端-服务器型数据库 | - |
| **部署方式** | 零部署，一个文件即可 | 需要安装服务端、配置用户、维护进程 | ✅ SQLite |
| **连接方式** | 直接读写本地文件 | TCP/IP 网络连接 | ✅ SQLite |
| **并发模型** | 单写多读（文件级锁） | 行级锁，支持高并发 | ✅ SQLite（个人使用） |
| **事务支持** | ACID 完整支持 | ACID 完整支持 | - |
| **Python 支持** | 内置 `sqlite3` 模块 | 需要安装 `pymysql`/`mysqlclient` | ✅ SQLite |
| **数据量级** | 万级到十万级 | 百万级到千万级 | ✅ SQLite |
| **适用场景** | 个人工具、嵌入式应用、小型项目 | 企业级应用、高并发系统、多用户场景 | ✅ SQLite |

**选择 SQLite 的理由：**

1. **零部署成本**：无需安装 MySQL 服务、配置用户权限、启动守护进程
2. **极简迁移**：整个数据库就是一个 `.db` 文件，备份=复制文件，迁移=拷贝文件
3. **性能足够**：本项目数据量级（触达用户、评论、任务记录）预计万级到十万级
4. **无网络开销**：单用户场景下，本地文件读写性能优于 MySQL 的网络连接
5. **依赖减少**：Python 内置 `sqlite3`，无需额外安装驱动
6. **渐进式扩展**：使用 SQLAlchemy ORM，未来迁移到 MySQL 只需修改连接配置

**渐进式架构策略：**
```python
# 当前：SQLite 连接
engine = create_engine('sqlite:///./automarketing.db')

# 未来：MySQL 连接（业务代码零改动）
engine = create_engine('mysql+pymysql://user:password@localhost/automarketing')
```

#### 4.2.4 SQLite 性能优化配置（PRAGMA）

```python
# SQLite 连接时启用优化配置
from sqlalchemy import create_engine

engine = create_engine(
    'sqlite:///./automarketing.db',
    connect_args={
        'check_same_thread': False,  # 允许多线程访问（FastAPI 异步场景）
    }
)

# 连接后执行 PRAGMA 优化
def configure_sqlite(conn):
    cursor = conn.cursor()
    # 启用 WAL（Write-Ahead Logging）模式，提升并发读写性能
    cursor.execute('PRAGMA journal_mode=WAL;')
    # 降低同步级别，提升写入性能（适用于非关键数据）
    cursor.execute('PRAGMA synchronous=NORMAL;')
    # 设置缓存大小（单位：页，每页约4KB，这里设置为64MB缓存）
    cursor.execute('PRAGMA cache_size=-16384;')
    # 启用内存映射（提升大查询性能）
    cursor.execute('PRAGMA mmap_size=1073741824;')  # 1GB
    # 启用查询优化器统计
    cursor.execute('PRAGMA query_only=FALSE;')
    # 设置临时文件存储（使用内存而非磁盘）
    cursor.execute('PRAGMA temp_store=MEMORY;')
    # 设置外键约束检查
    cursor.execute('PRAGMA foreign_keys=ON;')
    conn.commit()
```

**PRAGMA 配置说明：**

| 配置项 | 值 | 作用 |
|--------|-----|------|
| `journal_mode=WAL` | Write-Ahead Logging | 允许读操作和写操作同时进行，大幅提升并发性能 |
| `synchronous=NORMAL` | Normal | 写操作只在提交时同步到磁盘，平衡性能和安全性 |
| `cache_size=-16384` | -16384 | 设置页面缓存大小为 16384 页（约 64MB），负数表示使用内存 |
| `mmap_size=1073741824` | 1GB | 启用内存映射，大表查询性能提升明显 |

#### 8.19.6 登录流程改造

```
当前流程（Mock）：
  登录页 → mock接口返回token → 存储到localStorage → 跳转首页

目标流程（真实后端）：
  登录页 → POST /api/auth/login → 后端验证用户名密码
    → 成功：返回JWT token + 用户信息 + 角色 + 菜单权限
    → 前端存储token到localStorage
    → 前端存储用户信息/角色/权限到Pinia store
    → 根据菜单权限过滤侧边栏菜单
    → 跳转到首页
    → 失败：提示错误信息
```

## 九、核心业务操作流程

### 9.1 完整获客流程（以 TikTok/golf 为例）

```
步骤1: 配置基础数据
  配置中心 > 平台管理 > 确认 TikTok 平台已启用
  配置中心 > 业务线管理 > 确认 golf 业务线已创建
  配置中心 > 关键词管理 > 添加关键词 (golf simulator, indoor golf 等)
  配置中心 > 提示词管理 > 配置客户筛选提示词 + 私信生成提示词
  配置中心 > AI模型配置 > 配置并激活 DeepSeek 模型

步骤2: 创建爬虫任务
  任务中心 > 创建任务 > 选择「关键词爬虫」
  > 选择业务线: TikTok/golf
  > 勾选关键词: golf simulator, indoor golf
  > 启用AI筛选，选择「客户筛选-高尔夫」提示词
  > 设置最大采集数: 50条/关键词
  > 点击「立即启动」

步骤3: 监控任务执行
  任务中心 > 任务列表 > 查看任务状态
  > 点击「详情」查看实时进度
  > 日志面板实时展示: 搜索 -> 爬取评论 -> AI筛选
  > 筛选结果面板实时展示发现的潜在客户

步骤4: 查看筛选结果
  数据中心 > 触达用户 > 筛选状态: 待联系
  > 查看AI筛选出的潜在客户列表
  > 点击详情查看: 用户评论、来源视频、AI分析结果、生成的私信内容

步骤5: 创建私信任务
  任务中心 > 创建任务 > 选择「私信触达」
  > 选择业务线: TikTok/golf
  > 选择目标用户: 从筛选结果中勾选 (或全选)
  > 消息模式: 个性化生成 (使用「私信生成-高尔夫」提示词)
  > 选择账号: NEAGLE_GOLF
  > 设置发送上限: 50条, 间隔: 8-20分钟
  > 点击「立即启动」

步骤6: 监控私信发送
  任务中心 > 任务详情 > 实时查看发送进度
  > 成功/失败/待发送数量
  > 系统自动执行拟人化行为 + 冷却机制

步骤7: 数据统计
  统计分析 > 数据概览 > 查看完整转化漏斗
  > 采集 -> AI筛选 -> 私信发送 -> 回复/转化
```
